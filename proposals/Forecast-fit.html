ForecastFit

You have a smart outfit recommendations powered by real time forecasts.

Proposed by: Aayush

Description

ForecastFit is a weather outfit recommendation web application. Users enter their location or let the app detect it automatically and ForecastFit retrieves the current weather forecast. Based on temperature, rain, wind and other conditions it then suggests what to wear.

The idea is to close the gap between weather apps and daily wardrobe decisions. Most weather apps tell you it will be 9Â°C and cloudy but leave it to you to figure out what to wear. ForecastFit removes that hassle with simple, practical clothing guidance.

Registered users can build a virtual wardrobe, log the items they own, set style preferences (casual, formal, sporty etc.) and receive outfit suggestions based on what they actually own. The app can also learn from feedback like "too warm" or "too casual" to improve future suggestions.

User Stories

Guest

  • As a guest, I want to enter my city so I can see today's weather and a suggested outfit without an account.
  • As a guest, I want the app to detect my location automatically so I don't have to type it in.

Registered User

  • As a registered user, I want to create an account so my preferences are saved between sessions.
  • As a registered user, I want to set my style preference (casual, sporty, formal) so suggestions match my taste.
  • As a registered user, I want to add clothing items to my wardrobe so suggestions are based on what I own.
  • As a registered user, I want to give feedback on a suggestion so future suggestions improve.
  • As a registered user, I want to view a 7 day outfit plan so I can prepare outfits in advance.
  • As a registered user, I want a morning notification with today's outfit suggestion before I leave home.

Administrator

  • As an administrator, I want to add and edit weather thresholds so the recommendation engine stays accurate.
  • As an administrator, I want to manage user accounts to maintain platform safety.
  • As an administrator, I want to see usage statistics such as daily active users and popular locations.

Biggest Challenges

01

Recommendation Engine

Mapping weather conditions to outfit choices needs a flexible rule set that handles edge cases such as warm but raining.

02

Weather API Integration

Third-party APIs have rate limits. We will need caching to avoid repeated calls for the same location and handle API failures gracefully.

03

Cold Start Personalisation

New users have no wardrobe or feedback history. An onboarding questionnaire may be needed so first-time recommendations are still useful.

04

Geolocation & Privacy

The Geolocation API requires user permission and HTTPS. Location data must be handled in line with GDPR.

05

Team Coordination

Working on shared files risks merge conflicts. We will agree on a page structure early, use feature branches and make regular pull requests.

Data Requirements

The app will use a third party weather API (e.g. OpenWeatherMap or WeatherAPI.com) for temperature, precipitation, wind, humidity, UV index and a 7 day forecast. User profile data, wardrobe inventory and feedback must be stored securely with hashed passwords, in line with GDPR. API keys must stay server side only and never be exposed to the client.

K.P's Properties

Proposed by: Param and Kunj

Description

So, our idea is to design a web application for Real Estate, to manage viewing properties, filter prices, location and customer bookings etc. This system allows customers to search available properties, view detail information and submit booking request through this website. Admin can manage propertie listings, update information and monitor customer bookings through a structured management interface.

This application needs admin system so that they can add, delete and update the properties, prices, details and locations information.

Customers Public Users

  • Search Property Listings
  • Filter by Prices, Locations, Type
  • View Property Details
  • Submit Enquiry Form
  • Contact the Agency

  • Dashboard for Admin

    • Add New Property Listings
    • Edit Property Details
    • Delete Properties
    • Mark Property is sold or rented
    • View Enquires for Property
    • Manage Property Images

User Requirements-User Stories

As a visitor, I want to browse available tours so that I can choose a destination. As a visitor, I want to view detailed information about a tour so that I can understand the package. As a visitor, I want to submit a booking request so that I can reserve my trip. As a visitor, I want to send an enquiry so that I can ask questions.

Admin User Stories

As an admin, I want to add a new tour so that new travel packages can be offered. As an admin, I want to update tour details so that information stays accurate. As an admin, I want to delete tours so that outdated packages are removed. As an admin, I want to view bookings so that I can manage customer reservations.

Basic Data Requirements Structure

  • Properties Table: - id(Primary Key), title, description, price, location,property_type,bedrooms, bathroom.

  • Enquiries Table:- Id(Primary Key), Property_id(Foreign Key),name, email, phone,message.

  • Sole Selling Table:- Id(Primary Key), Property_id(Foreign Key), name, phone, final_price, cash, cheque.
